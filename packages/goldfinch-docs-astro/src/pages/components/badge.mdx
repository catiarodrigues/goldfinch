Displays a small label for status, categorization, or metadata.

## Installation

### Barrel

```tsx
import { Badge } from "@catiarodrigues/goldfinch";
```

### Granular

```tsx
import { Badge } from "@catiarodrigues/goldfinch/components/badge";
```

## Usage

```tsx
import { Badge } from "@catiarodrigues/goldfinch";

export default function Example() {
  return <Badge variant="secondary">New</Badge>;
}
```

### Color variants

Use these for product-specific labels where semantic colors don't fit.

<DemoFrame title="Color variants">
  <BadgeColorVariantsDemo />
</DemoFrame>

```tsx
<Badge variant="neutral">Neutral</Badge>
<Badge variant="red">Red</Badge>
<Badge variant="green">Green</Badge>
<Badge variant="orange">Orange</Badge>
<Badge variant="teal">Teal</Badge>
<Badge variant="blue">Blue</Badge>
<Badge variant="purple">Purple</Badge>
```

### Dot badges

Use `appearance="dot"` for a status indicator with a colored dot. Supported with `success`, `warning`, `error`, and `neutral` variants.

<DemoFrame title="Dot badges">
  <BadgeDotDemo />
</DemoFrame>

```tsx
<Badge appearance="dot" variant="success">Healthy</Badge>
<Badge appearance="dot" variant="warning">Warning</Badge>
<Badge appearance="dot" variant="error">Error</Badge>
<Badge appearance="dot" variant="neutral">Neutral</Badge>
```

### In a sentence

Badges sit inline with text for labeling items in context.

<DemoFrame title="In a sentence">
  <BadgeInSentenceDemo />
</DemoFrame>

```tsx
<p className="flex items-center gap-2">
      Users
      <Badge variant="secondary">New</Badge>
</p>
```

### Sizes

Three sizes are available to fit different contexts — `sm` for tight inline placement, `md` as the default, and `lg` for standalone emphasis.

<DemoFrame title="Sizes">
  <BadgeSizesDemo />
</DemoFrame>

```tsx
<Badge size="sm">SM</Badge>
<Badge size="md">MD</Badge>
<Badge size="lg">LG</Badge>
```
