Render headings, body copy, secondary text, monospace, and semantic color variants — each variant maps to the appropriate HTML element automatically.

## Installation

### Barrel

```tsx
import { Text } from "@catiarodrigues/goldfinch";
```

### Granular

```tsx
import { Text } from "@catiarodrigues/goldfinch/components/text";
```

## Usage

```tsx
import { Text } from "@catiarodrigues/goldfinch";

export default function Example() {
  return (
    <div>
      <Text variant="heading1">Heading 1</Text>
      <Text variant="body">Body copy goes here.</Text>
      <Text variant="secondary">Secondary / muted text.</Text>
      <Text variant="mono">npm install</Text>
    </div>
  );
}
```

Choose from variants like `heading1`, `heading2`, `heading3`, `body`, `secondary`, `mono`, etc. The component renders the corresponding HTML element.

```tsx
<Text variant="heading1">Page Title</Text>
<Text variant="heading2">Section Title</Text>
<Text variant="body">Regular paragraph text.</Text>
<Text variant="mono">Monospaced content</Text>
<Text variant="success">Saved successfully</Text>
<Text variant="error">Invalid input</Text>
```

<Callout type="info">
  The `variant` maps to semantic HTML elements — `heading1` renders `<h1>`, `body` renders `<p>`, and `secondary` renders `<p>` with muted styling.
</Callout>
